LoggingÔÉÅ

PipelineWise is generating log files at ~/.pipelinewise/<TARGET_ID>/<TAP_ID>/log from every run started by the run_tap command. Please check Running Pipelines section for further details how to run tap when you do a data sync.

Every log file will match the following pattern: <TARGET_ID>-<TAP_ID>-<DATE>_<TIME>.<SYNC_ENGINE>.log.<STATUS>

Variables:

  • TARGET_ID: Unique identifier of the target. This is the id property from a Targets (Destinations) YAML.

  • TAP_ID: Unique identifier of the tap. This is the id property from a Taps (Data Sources) YAML.

  • DATE: Date when the tap started in YYYYMMDD format

  • TIME: Time when the tap started in HH24MMSS format

  • SYNC_ENGINE: One of singer or fastsync. Check Replication Methods and Fast Sync section for further details.

  • STATUS: One of running, failed or success

Example:
